# Codex Note Rules

このリポジトリは、疑問を静的HTMLの質問ノートに変換して蓄積するための場所です。技術に限らず、料理、植物、生活知識、制度、買い物の調査など、あとで読み返したいテーマを扱います。

## Trigger

ユーザーの依頼が `質問:` または `質問：` で始まる場合は、1テーマ1ページの質問ノートを作成してください。

## Output Rules

- HTML/CSS/SVG/画像だけで構成し、ビルド工程を追加しない。
- 新規ノートはカテゴリ用ディレクトリの中に `index.html` として作る。
- 既存カテゴリが合わない場合は、短い英語の kebab-case でカテゴリディレクトリを作る。
- トップページ `index.html` に新規ノートへのリンクを追加する。
- 参照した一次情報、公式情報、信頼できる出典のURLを、各ノート末尾の `References` に必ず記載する。
- 秘密情報、個人情報、APIキー、トークン、社内情報、未公開情報は書かない。
- 出典が弱い内容は断定せず、本文中で不確実性を明示する。
- 医療、法律、金融、規制、食の安全、野草・植物の採取や摂食など、リスクのある内容では安全上の注意と専門家・公式情報確認の必要性を明記する。
- 画像を追加する場合は `images/` 配下に置き、相対パスで参照する。
- 生成後は `git diff` で確認し、ユーザーの明示承認なしに push しない。

## Page Style

- 読み返しやすい見出し、要約、図解、具体例を含める。
- 末尾に `Summary` と `References` を置く。
- 共通CSSは `styles/site.css` を使う。
- ページタイトル、説明、日付を明示する。

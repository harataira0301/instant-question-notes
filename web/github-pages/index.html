GitHub Pagesとは何か

GitHub上のファイルを、Webサイトとしてインターネットに公開できる仕組み。

Created: 2026-06-07

Overview

GitHub Pagesは、GitHubリポジトリに置いたHTML、CSS、JavaScript、画像などを、Webサイトとして公開するための機能です。個人サイト、プロジェクト紹介、ドキュメント、技術ノートの公開によく使われます。

このリポジトリのように、ビルド不要の静的HTMLを置いておけば、GitHub Pagesがそれを配信し、ブラウザからURLで読めるようにしてくれます。

ざっくりした流れ

何を公開できるか

GitHub Pagesが得意なのは、静的サイトです。静的サイトとは、アクセスされた時にサーバー側でデータベース処理やログイン処理を動かさず、あらかじめ用意したファイルをそのまま返すサイトです。

  • HTMLで書いたページ。
  • CSSで整えたデザイン。
  • JavaScriptで動くブラウザ内の処理。
  • 画像、PDF、フォントなどの静的ファイル。

公開URLの考え方

GitHub Pagesでは、リポジトリの種類や設定によってURLが決まります。プロジェクト用のサイトでは、一般的に次のような形になります。

https://ユーザー名.github.io/リポジトリ名/

このノート集なら、トップページはリポジトリ名までのURLで開き、個別ノートはその下のディレクトリとして開きます。

https://harataira0301.github.io/instant-question-notes/
https://harataira0301.github.io/instant-question-notes/web/github-pages/

GitHub Actionsとの関係

GitHub Pagesは、公開するファイルの用意にGitHub Actionsを使えます。Actionsは、pushなどをきっかけに自動で処理を走らせるGitHubの仕組みです。

このリポジトリでは、`main`ブランチへpushされると、GitHub Actionsがリポジトリの中身をPages用の成果物としてアップロードし、GitHub Pagesへデプロイします。つまり、HTMLを追加してpushすると、少し待ってから公開サイトに反映されます。

初心者向けの理解

GitHubを「ファイルを置く場所」だとすると、GitHub Pagesは「そのファイルをWebサイトとして見せる係」です。HTMLファイルをリポジトリに置くだけでは、基本的にはコード置き場にあるファイルです。Pagesを有効にすると、そのファイルがブラウザで見られる公開ページになります。

ブログサービスのように記事編集画面が用意されているわけではありません。その代わり、ファイルを直接管理でき、変更履歴がGitに残り、静的なノートやドキュメントを軽く公開できます。

Summary

GitHub Pagesは、GitHubリポジトリの中身を静的Webサイトとして公開する機能です。HTML、CSS、JavaScript、画像のようなファイルを置いてpushすると、GitHub Actionsなどのデプロイ処理を経て、`github.io` のURLで読めるようになります。サーバー処理が必要なWebアプリには向きませんが、技術ノートやドキュメント公開には扱いやすい仕組みです。

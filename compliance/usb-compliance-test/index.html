USBのコンプライアンステストとは何か

USB製品が仕様どおり動き、他社製品とも安全に接続できるかを確認する試験。

Created: 2026-06-13

Overview

USBのコンプライアンステストは、USB製品がUSB仕様に沿って実装されているかを確認する試験です。単に「PCにつながった」だけではなく、電気的な波形、プロトコル、デスクリプタ、電源、コネクタ、ケーブル、相互接続性などを確認します。

USB-IFのCompliance Programでは、実製品で適合性を測るために複数の試験仕様とTest ID、つまりTIDを使います。合格した製品はUSB-IF認証品として扱われ、Integrators Listへの掲載やUSB-IFロゴ利用につながります。

何のためにやるか

  • 他社のホスト、デバイス、ハブ、ケーブルと安定して接続するため。
  • 信号品質が規格範囲に入り、通信エラーや認識不良を減らすため。
  • Type-CやUSB PDで、電源供給・ロール切替・ケーブル判定を安全に行うため。
  • USB-IFロゴを製品やパッケージで使うため。
  • 量産後に「特定PCだけ認識しない」「特定ケーブルだけ充電できない」といった問題を減らすため。

USB 2.0で見ること

USB-IFのUSB 2.0試験情報では、Compliance Programの主な領域としてFunctional、Electrical、Interoperabilityが示されています。FunctionalではUSB30CVなどのツールで標準要求やクラス試験を確認し、Electricalでは物理層の信号品質を測り、Interoperabilityでは既知の良品USB製品との相互接続性を確認します。

USB 2.0 High-Speedでは、D+ / D-の差動配線、インピーダンス、スタブ、ESD保護部品、コネクタ周辺のレイアウトが波形に効きます。Full-Speedだけの製品でも、デスクリプタやサスペンド電流などで落ちることがあります。

USB Type-C / USB PDで見ること

Type-Cでは、従来のUSB信号だけでなく、コネクタの向き、CCピン、電源ロール、ケーブル検出、VCONN、Alternate Modeなどが増えます。USB PD対応製品では、PDメッセージ、電源契約、Source/Sink動作、保護動作も試験対象になります。

USB-IFのType-Cページでは、Type-C Functional Test Specification、Type-C/PD Source Power Requirements、USB PD Compliance Test Specificationなどが挙げられています。PD capable製品では、PD CTSのPhysical Layer、Protocol、Power Supplyの各セクションが関係します。

認証までの流れ

  1. 対象仕様を決めるUSB 2.0、USB 3.2、USB4、Type-C、USB PD、ケーブル、ハブなど、製品が主張する機能を明確にする。
  2. 試験マトリクスを確認する対象製品タイプに必要な試験項目、ツール、治具、認定機器を確認する。
  3. 社内で事前評価する波形、デスクリプタ、PDログ、電源シーケンス、ケーブル組み合わせを量産前に潰す。
  4. USB-IFへ製品登録する製品情報と必要資料を登録し、試験可能なAuthorized Independent Test Labを選ぶ。
  5. 試験所またはWorkshopで試験する認定試験所、USB-IF Sponsored Compliance Workshop、PILなどのルートを使う。
  6. USB-IFレビューを受ける試験データがレビューされ、合格すると認証・リスト掲載・ロゴ利用へ進む。

認証ルート

USB-IFは、製品認証のルートとして、Compliance Workshop、Authorized Independent Test Lab、Qualification by Similarity Program、Platform Interoperability Lab、OEM Arrangementsを挙げています。

初回製品や新規設計では、認定試験所での正式試験が基本になります。既存認証品と非常に近い派生品なら、SimilarityやOEMの仕組みを検討できる場合があります。ただし、どこまでが「重要な差分」かは慎重に判断され、最終的にはUSB-IF側の判断が関わります。

設計段階の注意点

  • USBロゴを使うなら、認証とUSB-IF Trademark License Agreementが必要になる。
  • USB-IF認証を目指す製品では、USB-IF認証済みコネクタを使う必要がある。
  • Type-Cは「コネクタをType-Cにしただけ」では済まず、CC制御と電源動作が重要。
  • PDはプロトコルだけでなく、電源の立ち上がり、保護、電圧精度、ケーブル能力も見る。
  • ESD保護部品やコモンモードチョークは、保護には効くが信号品質を悪化させることがある。
  • 試験治具、オシロスコープ、プロトコルアナライザ、PDテスタの準備に時間がかかる。
  • 量産版と試験サンプルの差分が大きいと、再試験や認証取り消しのリスクがある。

社内プリチェック

正式試験の前に、次のような確認をしておくと手戻りを減らせます。

USB 2.0/3.x:
- 差動インピーダンスと配線長
- アイパターン、ジッタ、振幅
- デスクリプタとVID/PID
- サスペンド/レジューム
- 複数ホスト・ハブ・ケーブルでの接続

USB Type-C/PD:
- CC抵抗、向き反転、VCONN
- Source/Sink/DRPのロール動作
- PDメッセージログ
- 電源契約後の電圧・電流
- 異常時の保護動作

Summary

USBのコンプライアンステストは、USB製品が仕様どおりに動き、他社製品とも安全に接続できることを確認する試験です。USB 2.0では主に機能、電気、相互接続性を見ます。Type-CやUSB PDでは、CC、電源ロール、ケーブル、PDプロトコル、電源品質まで範囲が広がります。ロゴ利用や公式認証を目指すなら、USB-IFの最新Compliance Updates、試験マトリクス、認定試験所の要件を早い段階で確認することが重要です。
